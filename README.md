## 1. top 명령어

### 📌 개요
`top`은 시스템의 **실시간 프로세스 상태**를 모니터링하는 명령어입니다. CPU 사용률, 메모리 사용량, 실행 중인 프로세스 정보를 동적으로 보여줍니다.



### ⌨️ 주요 옵션 및 단축키

| 옵션/키 | 설명 |
|---------|------|
| `top -d 5` | 5초마다 화면 갱신 |
| `top -u username` | 특정 사용자의 프로세스만 표시 |
| `top -p PID` | 특정 PID만 모니터링 |
| `q` | top 종료 |
| `k` | 프로세스 종료 (kill) |
| `M` | 메모리 사용량 순으로 정렬 |
| `P` | CPU 사용량 순으로 정렬 |
| `h` | 도움말 표시 |
| `1` | CPU 코어별로 표시 |

🎯 활용

- 시스템 성능 저하의 원인이 되는 프로세스를 즉시 식별할 때 유용합니다.
- top 화면 내에서 프로세스에 시그널을 보내 종료하거나(k 키), 우선순위를 변경(r 키)할 수 있습니다.
- 실시간 모니터링이 필요한 서버 관리 및 성능 분석에 필수적입니다.


## 2. ps 명령어

### 📌 개요
ps는 현재 실행 중인 프로세스의 스냅샷을 보여주는 명령어입니다. top이 실시간이라면, ps는 특정 순간의 정지된 목록입니다.



### ⌨️ 주요 옵션 및 단축키

| 옵션/키 | 설명 |
|---------|------|
| ps | 현재 터미널의 프로세스만 표시 |
| ps- e 또는 ps -A | 모든 프로세스 표시 |
|  ps -f| Full format으로 자세히 표시 |
| ps -ef | 모든 프로세스를 자세히 표시( unix스타일) |
| ps aux | 모든 프로세스를 자세히 표시 (BSD 스타일) |
| ps -u username | 특성 사용자이 프로세스 표시 |
| ps -p PID |특정 PID의 프로세스 정보 표시 |
| ps -C command | 특정 명령어 이름으로 검색 |
| ps --sort=-%mem | 메모리 사용량 내림차순 정렬 |
| ps --sort=-%cpu | CPU 사용량 내림차순 정렬 |

🎯 활용
- 특정 프로세스의 PID를 확인하여 kill 명령어의 대상으로 사용합니다.
- 프로세스의 실행 상태를 점검하고 시스템 리소스 사용량을 분석합니다
- 스크립트에서 프로세스 존재 여부를 확인할 때 유용합니다

## 3. jobs 명령어

### 📌 개요
jobs는 현재 셸(shell)에서 백그라운드 또는 정지 상태로 실행 중인 작업(job)들의 목록을 보여줍니다. 여기서 '작업(job)'은 사용자가 셸에서 시작한 명령어 또는 명령어 파이프라인을 의미합니다.


| 옵션/키 | 설명 |
|---------|------|
| jobs | 작업 목록 표시 |
| jobs -l| PID와 함께 표시 |
|  ps -f| Full format으로 자세히 표시 |
| jobs -p | PID만 표시 |
| jobs -r | 실행 중인 작업만 표시 |
| jobs -s | 정지된 작업만 표시 |
| jobs -n | 마지막 알림 이후 변경된 작업만 표시 | 

🎮 작업 제어 명령어

| 명령 | 설명 |
|---------|------|
| command & | 명령어를 백그라운드로 실행 |
| Ctrl + Z| 포어그라운드에서 실행 중인 작업을 정지 상태로 만듦 |
|  bg| 정지된 작업을 백그라운드에서 계속 실행되도록 함 |
| bg %N | 정지/백그라운드 작업 N을 백그라운드에서 재개 |
| fg | 백그라운드 작업을 포어그라운드로 가져옴 |
| fg %N | 정지/백그라운드 작업 N을 다시 포어그라운드로 가져옴 |
| %작업번호 |특정 작업 참조 |
| %% 또는 %+ | 현재 작업 참조 |
| %- | 이전 작업 참조 |

🎯 활용

- 여러 명령을 동시에 실행하며 터미널 사용의 효율을 높일 때 사용합니다.
- 특히 장시간 소요되는 작업을 백그라운드에서 실행하고 관리할 때 유용합니다.
- 편집 중인 파일을 잠시 두고 다른 작업을 할 때 Ctrl+Z로 정지했다가 fg로 다시 돌아올 수 있습니다.

⚠️ 주의사항

중요: jobs 명령어는 현재 셸 세션의 작업만 표시합니다. 다른 터미널이나 셸에서 실행한 작업은 보이지 않습니다.


Tip: 셸을 종료하면 백그라운드 작업도 함께 종료됩니다. 로그아웃 후에도 계속 실행하려면 nohup 명령어를 사용하세요.


## 4. kill 명령어

### 📌 개요
kill은 특정 프로세스에 시그널(Signal)을 보내 제어하는 명령어입니다. 시그널은 프로세스에게 특정 이벤트를 알리는 소프트웨어 인터럽트입니다.

### 기본 사용법
kill [옵션] PID

kill -시그널 PID

kill -시그널번호 PID

📋 주요 시그널

| 시그널 | 번호 |  설명 | 사용예시 |

|  bg| 정지된 작업을 백그라운드에서 계속 실행되도록 함 |
| bg %N | 정지/백그라운드 작업 N을 백그라운드에서 재개 |
| fg | 백그라운드 작업을 포어그라운드로 가져옴 |
| fg %N | 정지/백그라운드 작업 N을 다시 포어그라운드로 가져옴 |
| %작업번호 |특정 작업 참조 |
